# Juice Installer Workspace

This directory contains the macOS package installer assets.

## Layout

- `build/`: generated component package artifacts (ephemeral)
- `deps/`: third-party dependency `.pkg` files to include offline
- `payload/app/Applications/`: staging location for `Juice.app`
- `payload/scripts/usr/local/juice/`: runtime helper scripts/config
- `scripts/app/`: installer scripts for the app component package
- `scripts/scripts/`: installer scripts for the runtime component package
- `resources/`: optional installer resources shown by Installer.app
- `Distribution.xml`: generated by `scripts/build_pkg.sh`

## Build

Run:

```bash
./scripts/build_pkg.sh
```

Optional env vars:

- `PKG_SIGN_IDENTITY`: Developer ID Installer identity name
- `APP_SIGN_IDENTITY`: Developer ID Application identity name
- `UNSIGNED=1`: build unsigned pkg (internal testing)
- `NOTARIZE=1 NOTARY_PROFILE=<profile>`: submit and staple notarization
- `MIN_OS_VERSION=14.6`: minimum macOS installer target

## Sparkle Note

Juice Sparkle updates are distributed as signed/notarized `.app` archives (`.zip`) and do not use the `.pkg` installer channel. The installer scripts in this folder remain for offline/bootstrap installation only.

## Notarize Separately

After building a signed installer pkg, run:

```bash
NOTARY_PROFILE=<keychain-profile> ./scripts/notarize_pkg.sh
```

Or target a specific pkg:

```bash
NOTARY_PROFILE=<keychain-profile> ./scripts/notarize_pkg.sh /absolute/path/to/Juice-Installer-<version>.pkg
```

## Preflight Dependency Packages

Before building/notarizing the top-level installer, scan bundled dependency packages:

```bash
./scripts/check_pkg_notary_readiness.sh
```

You can also scan a single package:

```bash
./scripts/check_pkg_notary_readiness.sh /absolute/path/to/dependency.pkg
```
